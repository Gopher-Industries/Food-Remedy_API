# 👩‍💼 Leadership Guide

This document explains the responsibilities of leaders in the **Food Remedy** project.  
It is also a guide to help new leaders during the onboarding process.  
This guide began in **2025 Trimester 2**.  
The goal is to collect knowledge and experiences from past leaders so that future leaders can learn and succeed more easily.  

<br />

> 👉 The most important skill of a leader is **communication**.  

Your job is to make sure everyone stays on track. This includes communication with:  
- Members of your team  
- Other team leaders  
- Your mentor  
- The product owner and/or Company Director

## 🚀 Onboarding

### 💬 Create a Group Chat
- Create a group chat for all **Food Remedy members**.  
- This will be the **main place** for communication between members and the mentor.  
- In the first weeks, check the **team allocation file** from teaching staff.  
- Add members using their **student ID** (this is often easier than searching by name).  
- Add the **mentor** and any relevant teaching staff (e.g. whoever maintains the repository [Morgaine current as of 2025]).  
- Always include **chat history** when adding new members.  
- ⚠️ Do not use this chat for storing files. It may be deleted at the end of the trimester.

### 👥 Team Allocation
The best way to allocate students to teams is with a **Google Form + Google Sheet**.  
This also helps collect **GitHub usernames and emails** (needed for repository access).  

#### Suggested Questions
- Student ID  
- Name  
- GitHub Username  
- GitHub Email  
- *Optional:* Are you aiming for a High Distinction (HD)?  
- *Optional:* Which team would you most like to join?  
  - Frontend  
  - Backend  
  - Database  
  - Research  
  - *Add more teams if needed*  
- *Optional:* Are you interested in a leadership role this trimester or next?  
- *Optional:* Do you have strong GitHub experience?  

#### After Collecting Info
- By **week 2**, gather all GitHub usernames and emails.  
- Contact the person maintaining the repositories (e.g. Morgaine).  
- Politely ask them to add all users to the repository / gopher team.

## 💻 Contributions
Once everyone has access, the next step is teaching them **how to contribute**.  

- Point members to the **contribution documentation**.  
- If several members ask the same question, make another post to remind everyone.  
- Direct them to **online GitHub resources** if needed.  

⚠️ It is **not your job** to teach Git basics. This is a 3rd-year subject, and members are expected to know basic commands.  

### Minimum Git Commands
Make sure members know these commands:  

```bash
git clone "foodremedy-repository-link.git"
git checkout -b "branch-name"
git add .
git commit -m "message"
git push -u origin "branch-name"
git checkout main
git pull
```

## 📅 Scheduling Meetings
Leaders must schedule regular meetings with the product owner and company director.
- Ask your mentor about the correct timing for these meetings.
- Schedule recurring meetings at the same time each week.
- Post meeting times well before deadlines so members can plan.
- At the end of each meeting, confirm the next one.
- Create a Teams meeting link and share it in the group chat.

## 📖 Documentation
Documentation exists for your benefit and for future leaders.
- Read as much documentation as you can before the trimester begins.
- Update and maintain documentation regularly.
- Draft requirements for your trimester, outlining what each team should achieve.
- At the end of the trimester, work with juniors who will step into leadership roles.
- Begin writing draft tickets for the next trimester to help the transition.
